Introduction

In this report, we will solve the first part of the project for the Optimization and Decision course. We will address the CVRP (Capacitated Vehicle Routing Problem) using the Simplex Method. For this problem to be compatible with the Simplex Method, we must simplify it and formulate it as a Linear Programming (LP) problem.

The Capacitated Vehicle Routing Problem (CVRP) is a classic optimization problem in logistics and operations research. It involves determining the most efficient routes for a fleet of vehicles to deliver goods to a set of destinations, while considering capacity constraints on each vehicle. Each route visits a subset of nodes and starts and terminates at the depot (here represented by node 1).

The objective is to minimize the total distance or cost of all routes while ensuring that the demands of all destinations are met without exceeding the truck capacities, withouth specifying the number of trucks to be used.

Defining the Problem

For it to be seen as a Linear Programming (LP) Simplex problem we must find the expression for the value that we intend to minimize and the problem constraints. Those are the following:

Let xkij be a binary decision variable defined to indicate whether the vehicle k, k {1, 2, ..., p}, traverses an arc (i, j) in an optimal solution (1 if it is, 0 if it is not), cij representing the travelling distance from node i to node j and qj Q depicting the demand at destination node j. Each vehicle has a maximum capacity Q in terms of the total demand it can serve. The minimum number of vehicles needed to serve all customers is $$ \left\lceil \frac{\sum_{i=1}^{n} q_{i}}{Q} \right\rceil $$ Although this number does not necessarily represent the optimal number of vehicles required to satisfy the goal of this study.

The objective function is to minimize the total distance of each route while covering the demands of every node/costumer:

$$ \sum_{k=1}^{p}\sum_{i=1}^{n}\sum_{j=1,i\neq j}^{n} c_{ij} \: x_{kij} $$

subject to the following constraints:

Capacity Constraint

  1. The sum of the demands of the nodes visited in a route is less than or equal to the capacity of the vehicle performing the service: $$ \sum_{i=1}^{n}\sum_{j=2,i\neq j}^{n} q_{j} \: x_{kij} \leq Q \quad , \quad \forall k \in \{1, \dots, p\} $$

Routing Constraint

  1. Each node is visited by exactly one vehicle:
    $$ \sum_{k=1}^{p}\sum_{i=1,i\neq j}^{n} x_{kij} = 1 \quad , \quad \forall j \in \{1, \dots, n\} $$

Depot Constraint

  1. Each vehicle must start from the depot node:
    $$ \sum_{j=2}^{n} x_{k1j} = 1 \quad , \quad \forall k \in \{1, \dots, p\} $$

  2. Each vehicle must terminate at the depot node:
    $$ \sum_{i=2}^{n} x_{ki1} = 1 \quad , \quad \forall k \in \{1, \dots, p\} $$

Flow Constraint

  1. The number of the vehicles arriving at every customer and entering the depot is equal to the number of the vehicles leaving:
    $$ \sum_{i=1,i\neq j}^{n} x_{kij} = \sum_{i=1}^{n} x_{kji} \quad , \quad \forall j \in \{1, \dots, n\}, \: k \in \{1, \dots, p\} $$

Obligatory Constraint

  1. Specify the definition domains of the variables: $$ x_{kij} \in \{0,1\} \quad , \quad \forall k \in \{1, \dots, p\}, \: i,j \in \{1, \dots, n\} $$

This model is known as a three-index vehicle flow formulation. The number of inequalities of the sub-route elimination constraints grow exponentially with the number of nodes.

Defining the cost matrix

The cost matrix for the problem in hand is a $n \times n$ matrix, being n the total number of nodes (destination nodes + depot). Each entry of the matrix displays the euclidean distance between two nodes. For that reason, this is a symmetric matrix with zero diagonal.

This matrix can be computed using the values of X and Y coordinates of each node, provided in the datasets used.

def calculate_distance(coordinates, i, j):
    """
    Calculate the Euclidean distance between two nodes.
    """
    x1, y1 = coordinates[i]
    x2, y2 = coordinates[j]
    return np.sqrt((x1 - x2) ** 2 + (y1 - y2) ** 2)


def calculate_distance_matrix(coordinates):
    """
    Calculate the distance matrix between coordinates.
    """
    num_points = len(coordinates)
    dist_matrix = np.zeros((num_points, num_points))

    for i in range(num_points):
        for j in range(num_points):
            dist_matrix[i, j] = calculate_distance(coordinates, i, j)

    return dist_matrix

def calculate_total_distance(route, dist_matrix):
    """
    Calculate the total distance of a given route using the distance matrix.
    """
    total_distance = 0
    num_points = len(route)

    for i in range(num_points - 1):
        current_node = route[i]
        next_node = route[i + 1]
        total_distance += dist_matrix[current_node, next_node]

    return total_distance

    model.K = pyEnv.Set(initialize=range(n_vehicles))

    # Decision variables xijk and yik
    model.x = pyEnv.Var(model.A, model.K, within=pyEnv.Binary)
    model.y = pyEnv.Var(model.V, model.K, within=pyEnv.Binary)

    # Demand for each node
    model.demand = pyEnv.Param(model.V, initialize={i: d for (i, d) in enumerate(demand)})

    # Vehicle capacity
    model.capacity = pyEnv.Param(initialize=capacity)

    # Cost Matrix cij
    model.c = pyEnv.Param(model.A, initialize={(i, j): cost_matrix[i, j] for (i, j) in model.A})

    # Objective Function
    model.objective = pyEnv.Objective(
        expr=sum(
            model.x[i, j, k] * model.c[i, j]
            for (i, j) in model.A
            for k in model.K
        ),
        sense=pyEnv.minimize,
    )

    def arcs_in(model, i):
        if i == model.V.first():
            return sum(model.x[:, i, :]) == len(model.K)

            return sum(model.x[:, i, :]) == 1.0

            return sum(model.x[i, :, :]) == len(model.K)
        else:
            return sum(model.x[i, :, :]) == 1.0
